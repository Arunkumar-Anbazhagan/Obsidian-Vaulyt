## üçΩÔ∏è What is an API?
* **[00:00:33](https://youtu.be/y_cpFMF1pzk?t=33)**: **The Restaurant Analogy**:
    * **You (Client)**: Sitting at the table.
    * **Waiter (API/Interface)**: Takes your order to the kitchen and brings food back.
    * **Kitchen (Application)**: Prepares the food (data/service).
    * This analogy explains how an API acts as an intermediary.
* **[00:01:31](https://youtu.be/y_cpFMF1pzk?t=91)**: **Technical Definition**: API stands for **Application Programming Interface**. It exposes a service that developers write programs to consume (e.g., Google Sheets API allowing you to read/write data).
* **[00:02:53](https://youtu.be/y_cpFMF1pzk?t=173)**: **Why do we need it?**: To **abstract complexity**. Instead of accessing a server's database directly (going to the kitchen), you use a simple interface (asking the waiter).

## ‚öôÔ∏è How an API Works
* **[00:03:48](https://youtu.be/y_cpFMF1pzk?t=228)**: **Documentation**: Acts like a menu, explaining how to use the API.
* **[00:04:10](https://youtu.be/y_cpFMF1pzk?t=250)**: **Client-Server Model**:
    * **Request**: Sent by the Client (You) through the Interface.
    * **Response**: Sent back by the Server (Application).

### üì§ Components of an HTTP Request
* **[00:04:53](https://youtu.be/y_cpFMF1pzk?t=293)**: The 4 main components of a request:
    1.  **URL** ([00:05:35](https://youtu.be/y_cpFMF1pzk?t=335)): Unique location for a resource (Scheme, Host, Path, Query Parameters).
    2.  **Method** ([00:06:15](https://youtu.be/y_cpFMF1pzk?t=375)): The action to perform.
        * `GET`: Receive information (Reading data).
        * `POST`: Send information (Submitting a form).
    3.  **Header** ([00:07:26](https://youtu.be/y_cpFMF1pzk?t=446)): Context for the request (e.g., location, language, device type, content format like `application/json`).
    4.  **Body** ([00:08:22](https://youtu.be/y_cpFMF1pzk?t=502)): Optional (for POST requests). Contains the actual data being sent to the server.

* **[00:08:58](https://youtu.be/y_cpFMF1pzk?t=538)**: **Credentials**: How you prove authorization to the server.
    * **API Key**: In the request body , Often sent via query parameters or headers.
    * Bearer Token: inside the header 
    * **OAuth**: "Sign in with Google" style authentication.

### üì• Components of an HTTP Response
* **[00:10:28](https://youtu.be/y_cpFMF1pzk?t=628)**: The 3 main components of a response:
    1.  **Status Code** ([00:10:37](https://youtu.be/y_cpFMF1pzk?t=637)): A 3-digit number indicating success or failure.
        * `200` (OK): Success.
        * `401` (Unauthorized): Authentication problem.
        * `404` (Not Found): URL or resource not found.
        * `500` (Internal Server Error): The server had a problem.
    2.  **Header** ([00:12:34](https://youtu.be/y_cpFMF1pzk?t=754)): Context about the response (content length, type, expiration).
    3.  **Body** ([00:12:58](https://youtu.be/y_cpFMF1pzk?t=778)): The actual data returned (HTML, JSON, etc.).

## üîî Webhooks (Reverse APIs)
* **[00:13:23](https://youtu.be/y_cpFMF1pzk?t=803)**: **The Doorbell Analogy**:
    * Instead of checking the door every few minutes (**Polling**), you wait for the doorbell to ring (**Webhook**).
* **[00:13:56](https://youtu.be/y_cpFMF1pzk?t=836)**: **Polling vs. Webhooks**:
    * **Polling**: Asking "Is there new data?" repeatedly.
    * **Webhook**: The service sends data automatically when an event happens (e.g., a new payment in Stripe).
* **[00:14:57](https://youtu.be/y_cpFMF1pzk?t=897)**: To set up a webhook, you need a **URL** to send the information to (which n8n provides).
