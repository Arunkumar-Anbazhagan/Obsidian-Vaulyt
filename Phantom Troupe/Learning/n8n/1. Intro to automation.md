* **[00:00:59](https://youtu.be/4BVTkqbn_tY?t=59)**: **Data-Driven Decisions**: Automation moves away from intuition/feeling (subjective) to data-driven approaches (objective/logical).

## ‚öôÔ∏è What is Automation?
* **[00:02:34](https://youtu.be/4BVTkqbn_tY?t=154)**: **Definition**: A predictable set of predetermined actions that transfers data from one point to another.
* **[00:02:52](https://youtu.be/4BVTkqbn_tY?t=172)**: **Example Workflow**:
    1.  **Trigger**: Form submission.
    2.  **Logic**: Check company type.

        * *High Value*: Add to Google Sheet.
        * *Ideal Customer*: Notify Account Manager ASAP.

## üß± Core Concepts of Automation
* **[00:04:45](https://youtu.be/4BVTkqbn_tY?t=285)**: Introduction to the three main building blocks.

### 1. Triggers
* **[00:04:52](https://youtu.be/4BVTkqbn_tY?t=292)**: What starts the automation. It has an exit arrow but no entry arrow.
* **Types**:
    * **Manual**: Executed by a person.
    * **Scheduled**: Specific times (e.g., every day at 8 AM).
    * **App Events**: Webhooks, CRM updates, Form submissions.

### 2. Filtering (Logic)
* **[00:05:52](https://youtu.be/4BVTkqbn_tY?t=352)**: Allows you to allow or block data based on specific conditions (e.g., Company Value).
* Using filters allows data to follow specific paths.

### 3. Actions (Apps)
* **[00:06:55](https://youtu.be/4BVTkqbn_tY?t=415)**: Interaction with web applications.
* **Examples**:
    * **Google Sheets**: Update rows, create sheets.
    * **Dropbox**: Upload/move files.
    * **Slack**: Send messages, get user info.
    * **Salesforce**: Create leads, get contacts.

## üîÑ What is a Workflow?
* **[00:07:53](https://youtu.be/4BVTkqbn_tY?t=473)**: Most workflows follow a standard template:
    1.  **Trigger**: Launches the workflow.
    2.  **Process**: Sorting, filtering, formatting, or transforming data ("If this then that").
    3.  **Action**: The final output (e.g., sending an email, updating a sheet).

## ‚úÖ Best Practices
* **[00:09:03](https://youtu.be/4BVTkqbn_tY?t=543)**: **Map it out first!** Do not start building immediately.
    * **Benefits of Mapping**:
        * Ensures understanding of the task.
        * Provides visibility on tools/apps needed.
        * Confirms feasibility (prevents wasted time).
        * Helps estimate workload.
        * Identifies where human intervention might still be needed.

### How to Map an Automation
* **[00:10:46](https://youtu.be/4BVTkqbn_tY?t=646)**: Use tools like Miro or FigJam.
    1.  Create a flowchart.
    2.  List every part of the process as a block.
    3.  Link blocks with arrows (Left to Right).
    4.  Verify logic before building in n8n.
