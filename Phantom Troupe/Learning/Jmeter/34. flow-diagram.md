# Flow diagram   
Thread Group
├─ Configuration Elements
│    ├─ HTTP Request Defaults
│    ├─ CSV Data Set Config
│    └─ Cookie/Header/Cache Managers
│
├─ Pre-Processors
│    ├─ JSR223 PreProcessor
│    ├─ User Parameters
│    └─ URL Rewriting Modifier
│
├─ Timers
│    ├─ Constant Timer
│    ├─ Uniform Random Timer
│    └─ Constant Throughput Timer
│
├─ Samplers
│    ├─ HTTP Request
│    ├─ JDBC Request
│    ├─ SOAP/XML-RPC Request
│    └─ Custom Sampler
│
├─ Post-Processors
│    ├─ Regex Extractor
│    ├─ JSON Extractor
│    └─ XPath Extractor
│
├─ Assertions
│    ├─ Response Assertion
│    ├─ Duration Assertion
│    └─ Size Assertion
│
└─ Listeners
├─ View Results Tree
├─ Summary Report
├─ Aggregate Report
└─ Graph Results   
“JMeter executes elements in a strict order: **Config → Pre‑Processor → Timer → Sampler → Post‑Processor → Assertion → Listener**. This ensures requests are parameterized, modified, paced, executed, validated, and finally reported.”
