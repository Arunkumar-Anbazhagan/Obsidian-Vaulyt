# Topic: The Lifecycle of a Resource (CRUD)

## Authentication (Bearer Token)
1.  **Register Client:** POST to `/api-clients` to get an `accessToken`.
2.  **Set Auth:** Use **Authorization Tab** -> **Type: Bearer Token** -> **Token: {{accessToken}}**.

## Operations
* **POST /orders:** Creates an order. Requires Authorization and a JSON body.
    * **Success:** `201 Created`. Response contains `orderId`.
* **PATCH /orders/:orderId:** Updates part of an existing order (e.g., customer name).
    * **Body:** Only include the fields to be changed.
    * **Success:** `204 No Content`.
* **DELETE /orders/:orderId:** Removes the resource.
    * **Success:** `204 No Content`. Verified by a subsequent `GET` returning 404.
