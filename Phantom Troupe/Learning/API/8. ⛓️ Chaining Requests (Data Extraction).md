# Topic: Passing Data Between Requests

## Key Concepts
* **Chaining:** Using the output of one request (e.g., an `orderId`) as the input for the next request.
* **Programmatic Variables:** Setting variables via the Tests script.

## Example: Extracting and Setting `orderId`
* **Location:** Tests tab of the `POST /orders` request.
```javascript
const response = pm.response.json();
// Save the dynamic orderId for later use
if (response.orderId) {
    pm.globals.set("orderId", response.orderId);
}
```

- **Usage in next request:** `GET /orders/{{orderId}}`

# Topic: Running Collections and Controlling Flow

## A. Collection Runner
* **Function:** Executes all requests in a collection sequentially, providing an aggregated pass/fail report for all assertions.

## B. Controlling Execution Order (Workflows)
* **Function:** Use `postman.setNextRequest()` to override the default linear flow.
* **Example (Skipping a request):**
```javascript
// Jump to a request named "List Books" instead of the next one
postman.setNextRequest("List Books"); 
// To stop the run: postman.setNextRequest(null);
````

````
## A. Postman Monitors (Cloud)
* **Function:** Schedules collection runs in the cloud for continuous API health checks.
* **Gotcha:** Monitors use the **Initial Value** of variables, not the Current Value. Secrets must be saved to the Initial Value to work in the cloud.
````
